# Testing and Definition of Done Rules

- Toute feature majeure doit avoir un test ou une preuve de test.
- Les parcours critiques doivent être couverts en E2E :
  - consulter l'annuaire
  - filtrer
  - ouvrir une fiche
  - soumettre un lieu
  - corriger une fiche
  - traiter une contribution en admin
- Gérer explicitement les états vides, erreurs, chargement et conflits.
- Une feature n'est pas done sans cohérence public/admin/data.
- Mettre à jour la documentation quand le comportement métier change.
