# Memory seed — Annuaire Dog Friendly 974

Ce fichier n'est pas destiné à rester dans le repo comme mémoire active.
Il sert de base à copier, si besoin, dans la mémoire projet de Claude Code via `/memory`.

## Ce que Claude doit garder en tête
- Produit = annuaire éditorial augmenté par la data.
- Ne jamais traiter une source externe comme vérité absolue.
- Toujours raisonner en `source + preuve + date + confiance`.
- Priorité produit : fiabilité, fraîcheur, administrabilité.
- L'admin est un poste de pilotage, pas un CRUD cosmétique.
- Le MVP doit rester frugal et web-first.
- Toute logique IA doit rester au service de l'extraction et de la structuration de faits explicites.
- Si une source n'est pas explicite sur l'accueil des chiens, le statut doit rester `unknown`.
- La donnée publique doit rester compréhensible en moins de 10 secondes.
- Le produit cible d'abord La Réunion, avec un vocabulaire et une UX utiles localement.

## Rappels de style
- Écrire court, précis, actionnable.
- Pas de jargon technique inutile dans le front public.
- Pas de ton corporate froid.
- Pas de ton gadget "cute dog app".
